ابنيلي تطبيق كامل اسمه "مندوبك" - منصة دليفري زي أوبر. استخدم "القلب النووي Trinity Stack" ونفذ كل القواعد الجاية بالحرف:

=== فكرة التطبيق بالتفصيل ===
1. نوعين يوزر: عميل + مندوب + ادمن
2. المندوب: يسجل باسمه، رقم موبايل، عنوان، صورة البطاقة وش وضهر. رقم الموبايل Unique مينفعش يتكرر. ليه صفحة بروفايل ونسبة تقييم.
3. العميل: يطلب أوردر. الأوردر نوعين: أ. مشتريات من مكان واحد أو أكتر ب. توصيل من عنوان A لـ عنوان B. العميل ميدفعش فلوس، بيدفع تمن التوصيل بس.
4. التسعير: أنا كـ ادمن بحدد سعر الكيلو/المنطقة في لوحة تحكم. أول ما العميل يدخل العناوين، التطبيق يحسب سعر التوصيل أوتوماتيك قبل ما يأكد الأوردر عشان المندوب مينصبش.
5. صفحة العروض/الإعلانات: دي صفحة الهوم. أنا كـ ادمن بنزل عروض مطاعم/محلات بمقابل مادي. لو العميل ضغط على عرض، التطبيق يسجله أوردر جاهز بالعنوان والمنتج ويقوله "تأكد الطلب؟".
6. لوحة المناديب: كل المناديب شايفة الأوردرات الجديدة Live. أول مندوب يدوس "قبول" الأوردر يتحجز باسمه ويختفي من عند الباقي في أقل من 0.1 ثانية. ممنوع مندوبين ياخدوا نفس الأوردر.
7. الأمان: ممنوع مندوب يسجل برقمين مختلفين. رقم الموبايل + صورة البطاقة = بصمة واحدة.

=== التكنولوجيا - القلب النووي ===
1. Frontend: Astro + TypeScript + Tailwind + Shadcn UI على Cloudflare Pages
2. Backend: Cloudflare Workers + Hono.js للـ API
3. Auth: Clerk - تسجيل بجوجل/رقم/OTP + 2FA + منع رقم موبايل مكرر
4. Database: Supabase Postgres + Prisma ORM
5. Cache L1: Cloudflare KV عشان السباق على الأوردرات
6. Storage: Cloudflare R2 لصور البطايق والمنتجات
7. Realtime: Supabase Realtime عشان الأوردر يختفي لحظياً من عند كل المناديب

=== القواعد الذهبية - دي اللي هتخليه أسطوري ===
1. منع السباق على الأوردر Race Condition: استخدم "Atomic Transaction" في Supabase + Cloudflare KV Lock. الكود: لما مندوب يدوس قبول، اعمل SELECT FOR UPDATE. لو الأوردر status=pending غيره لـ accepted وخزن id المندوب. لو status اتغير خلاص ارجعله "الأوردر اتحجز". ده لازم يكون في Transaction واحدة.

2. الأمان 100% RLS: 
   - جدول orders: اليوزر يشوف الأوردر بتاعه بس. المندوب يشوف الأوردرات اللي status=pending + الأوردرات بتاعته بس.
   - جدول couriers: المندوب يشوف بروفايله بس. الادمن يشوف الكل.
   - صور البطايق في R2: اللينك يكون Signed URL صالح 5 دقايق بس.

3. منع تكرار المندوب: في Clerk اعمل Action يمنع تسجيل نفس رقم الموبايل مرتين. وفي Supabase اعمل Unique Index على رقم الموبايل في جدول couriers.

4. السرعة 100%: أي طلب "جيب الأوردرات الجديدة" يتكاش في Cloudflare KV 3 ثواني بس. عشان المناديب تشوف الأوردر لحظي بس السيرفر ميقعش.

5. حساب السعر: اعمل جدول admin_pricing فيه [المنطقة, سعر_الكيلو, حد_أدنى]. API يحسب المسافة بين النقطتين بـ Haversine formula ويضرب × السعر + الحد الأدنى. النتيجة تظهر للعميل قبل الدفع.

6. الكود نضيف: فولدرات /api/orders /api/couriers /api/admin /components /lib/auth /lib/pricing

=== المطلوب منك دلوقتي ===
1. Prisma Schema كامل: جداول users, couriers, customers, orders, order_items, addresses, admin_pricing, ads_offers
2. RLS Policies كاملة SQL جاهزة للنسخ في Supabase
3. API Endpoints Hono: POST /orders, GET /orders/pending, POST /orders/:id/accept, POST /upload-id, GET /pricing/calculate
4. كود Cloudflare KV Lock للـ accept order عشان السباق
5. كود رفع صورتين البطاقة على R2 + يخزن اللينك
6. صفحة هوم فيها قسم العروض + صفحة طلب أوردر + لوحة المندوب Live
7. ملف .env.example بكل الـ Keys
8. ملف wrangler.toml
